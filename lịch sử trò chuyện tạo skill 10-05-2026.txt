  ┌─────────────┬────────────────────────────────────────────────┐

  ├─────────────┼────────────────────────────────────────────────┤

  ├─────────────┼────────────────────────────────────────────────┤
  │ Bước 0      │ Luôn load product-marketing-context trước      │
  ├─────────────┼────────────────────────────────────────────────┤
  │ Bước 1-N    │ Workflow từng bước cụ thể                      │
  ├─────────────┼────────────────────────────────────────────────┤
  │ Framework   │ Template, công thức, checklist copy-paste được │
  ├─────────────┼────────────────────────────────────────────────┤
  │ Output cuối │ Agent biết cần trả ra định dạng gì             │
  └─────────────┴────────────────────────────────────────────────┘

  # Cấu trúc chuẩn của một AI Marketing Skill                                   
                                                                                
  Một skill tốt thường gồm 7 phần:                                              
                                                                                
  ```md                                                                         
  ---                                                                           
  name: social-content                                                          
  description: Create high-converting social media content                      
  ---                                                                           
                                                                                
  # Purpose                                                                     
  Skill này dùng để...                                                          
                                                                                
  # When to use                                                                 
  Dùng khi user muốn...                                                         
                                                                                
  # Inputs needed                                                               
  - Product info                                                                
  - Audience                                                                    
  - Platform                                                                    
  - Goal                                                                        
                                                                                
  # Workflow                                                                    
  1. Analyze audience                                                           
  2. Identify hook                                                              
  3. Create content angle                                                       
  4. Generate CTA                                                               
  5. Optimize for platform                                                      
                                                                                
  # Frameworks                                                                  
  - AIDA                                                                        
  - PAS                                                                         
  - Hook-Rehook                                                                 
  - Storytelling                                                                
                                                                                
  # Output format                                                               
  ## Hook                                                                       
  ## Caption                                                                    
  ## CTA                                                                        
  ## Hashtags                                                                   
                                                                                
  # Best practices                                                              
  - Keep hook under 8 words                                                     
  - Use emotional trigger                                                       
  - Avoid generic CTA                                                           
  ```

❯ YAML frontmatter có nghĩa là gì                                               

● YAML frontmatter là phần metadata nằm ở đầu file markdown, được kẹp giữa hai
  dấu ---.

  ---
  name: image
  description: Create marketing images...
  ---

  # Nội dung bắt đầu từ đây

  ---
  YAML là gì?

  YAML = ngôn ngữ viết dữ liệu dạng key: value, dễ đọc hơn JSON.

  name: image                      # tên skill
  description: Create images...    # mô tả ngắn

  Frontmatter là gì?

  Front = phía trước, matter = nội dung → phần thông tin đặt ở đầu file, trước
  nội dung chính.

  ---
  Tại sao cần nó?

  Khi AI agent đọc file skill, nó cần biết nhanh:
  - File này tên gì? (name)
  - File này dùng để làm gì? (description)

  Thay vì phải đọc toàn bộ file để đoán, agent chỉ cần scan phần ---...--- ở đầu
   là đủ — giống như nhìn vào bìa sách thay vì đọc hết cuốn sách.

  So sánh thực tế

  ┌────────────────────┬───────────────────────┬────────────────────────────┐
  │                    │ Không có frontmatter  │       Có frontmatter       │
  ├────────────────────┼───────────────────────┼────────────────────────────┤
  │ Agent tìm skill    │ Phải đọc toàn bộ file │ Đọc 2 dòng là xong         │
  ├────────────────────┼───────────────────────┼────────────────────────────┤
  │ Quản lý nhiều file │ Phải mở từng file     │ Nhìn metadata là biết      │
  ├────────────────────┼───────────────────────┼────────────────────────────┤
  │ Tool hỗ trợ        │ Không nhận diện được  │ Tự động index, search được │
  └────────────────────┴───────────────────────┴────────────────────────────┘

❯ cách sử dụng YAML fronmatter trong terminal là gì                             

● YAML frontmatter không phải lệnh terminal — nó là phần nội dung bên trong file
   markdown. Bạn viết nó bằng text editor hoặc khi tạo file. Nhưng có nhiều cách
   dùng nó qua terminal:

  ---
  1. Đọc frontmatter bằng terminal

  Dùng tool yq (đọc YAML) hoặc Python:

  # Xem toàn bộ nội dung file
  cat .claude/skills/image.md

  # Dùng Python để chỉ đọc phần frontmatter
  python -c "
  import re
  content = open('.claude/skills/image.md', encoding='utf-8').read()
  match = re.search(r'^---(.+?)---', content, re.DOTALL)
  print(match.group(1))
  "

  ---
  2. Liệt kê tên và mô tả tất cả skills

  Lệnh thực tế hữu ích — đọc name và description từ tất cả file skill:

  python -c "
  import os, re

  skills_dir = '.claude/skills'
  for fname in os.listdir(skills_dir):
      if not fname.endswith('.md'): continue
      path = os.path.join(skills_dir, fname)
      content = open(path, encoding='utf-8').read()
      match = re.search(r'^---(.+?)---', content, re.DOTALL)
      if match:
          for line in match.group(1).strip().splitlines():
              print(f'{fname}: {line}')
      print()
  "

  ---
  3. Tạo file skill mới có frontmatter (từ terminal)

  # Gõ lệnh này trong PowerShell
  @'
  ---
  name: email-marketing
  description: Write email sequences, subject lines, and newsletters
  ---

  # Purpose
  ...
  '@ | Out-File -FilePath ".claude\skills\email-marketing.md" -Encoding utf8

  gì. Khi agent cần tìm skill phù hợp, nó scan phần ---...--- để biết skill nào
  làm gì, rồi load đúng file đó.
